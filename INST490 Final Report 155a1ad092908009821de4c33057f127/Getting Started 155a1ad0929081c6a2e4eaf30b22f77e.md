# Lakeland Census Data ETL Project

This repository contains an **ETL (Extract, Transform, Load)** pipeline for processing historical census data from Lakeland. The project transforms raw census data into a structured, normalized database format and loads it into a **Supabase PostgreSQL** database.

### **Features**

- Extracts raw census data from multiple sources and years.
- Transforms the data into a clean, structured format.
- Loads the data into a relational database for easy querying and analysis.

---

### **2. Database Schema**

---

# Database Schema

The database follows a **normalized relational structure** for better organization and scalability. It ensures clear relationships between tables and eliminates data redundancy.

### **Database Tables Overview**

Key tables in the database include:

- `census_records`: Core census metadata, such as year and source information.
- `persons`: Individual records with unique identifiers.
- `families`: Family unit details.
- `relationships`: Relationships between family members.
- `locations`: Address and dwelling information.
- `personal_attributes`: Demographics such as age, sex, and race.
- `occupations`: Employment-related data.
- `property_status`: Details of property ownership or rental.
- `marital_status`: Marital status information.

### **Step 2: Database Schema Setup**

1. Open [dbdiagram.io](https://dbdiagram.io/).
2. Copy the database schema from `ERD/dbdiagram.io_schema.dbml` into the editor.
3. In Supabase, open the **SQL Editor** and:
    - Create a new query.
    - Paste the schema from `ERD/dbdiagram.io/ERD.sql`.
    - Execute the query to create all tables.
4. Verify table creation in the **Table Editor**.

### **Step 3: Configure Environment Variables**

    
2. Update `.env.local` with your Supabase credentials:Credentials can be found in **Supabase Project Settings**.
    

# Running the ETL Pipeline

    ```bash

    Copy code
    ```

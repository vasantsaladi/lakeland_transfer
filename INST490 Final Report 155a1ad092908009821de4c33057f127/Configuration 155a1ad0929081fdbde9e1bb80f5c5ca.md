Proper configuration is vital for ensuring the **Lakeland Census Data ETL Project** operates efficiently, securely, and meets the specific needs of your environment. ## **Configuration Overview**

This project is designed with flexibility in mind, allowing you to tailor various aspects of the system to your requirements. Configuration options include:

- Environment Variables
- Database Settings
- API Keys and Third-Party Integrations
- Application Preferences

---

## **Environment Variables**

Environment variables control critical aspects of the application, such as database connections, API integrations, and application behavior. These should be set in a `.env.local` file or managed via your hosting provider's environment settings.

### **Key Environment Variables**

- **`SUPABASE_URL`**: The URL of your Supabase instance.
- **`SUPABASE_KEY`**: The API key for accessing your Supabase database.
- **`DATABASE_URL`**: (Optional) Connection string for direct database access.
- **`EMAIL_SETTINGS`**: Configuration for email notifications (e.g., SMTP server, port, user credentials).

### **Configuring Environment Variables**

1. Copy the example file:
    
    ```bash
    bash
    Copy code
    cp .env.example .env.local
    
    ```
    
2. Update `.env.local` with your specific credentials:
    
    ```
    env
    Copy code
    SUPABASE_URL=https://your-project.supabase.co
    SUPABASE_KEY=your-anon-key
    
    ```

## **Database Configuration**

The application uses a relational database to store processed census data. Proper database configuration ensures data integrity, performance, and scalability.

### **Key Database Settings**

- **Database Type**: The application uses PostgreSQL (via Supabase) for storage and querying.
- **Schema**: A normalized schema defines tables, relationships, and constraints.
- **Indexes**: Indexes are applied to improve query performance for large datasets.
- **Constraints**: Foreign keys and other constraints ensure data integrity.

### **Setting Up the Database**

1. Use the SQL script provided in the repository to initialize the schema.
    
    ```sql
    sql
    Copy code
    -- Example of creating a table
    CREATE TABLE census_records (
      record_id SERIAL PRIMARY KEY,
      census_year INT NOT NULL,
      source_pk INT NOT NULL
    );
    
    ```
    
2. Verify the schema in the Supabase **Table Editor** after execution.

The application can integrate with third-party services to extend functionality. Configure these services in the appropriate settings files or through environment variables.

- **Supabase**: The database backend for storage and querying.
- **Email Services**: SMTP settings for email notifications.

## **Configuration Best Practices**

To ensure smooth operation and robust security, adhere to the following best practices:

### **1. Secure Sensitive Information**

- Use encrypted storage solutions for API keys and database credentials.
- Avoid hardcoding secrets in source code.

### **2. Use Version Control**

- Keep configuration files under version control but exclude sensitive files like `.env.local` using `.gitignore`.
    
    ```
    plaintext
    Copy code
    # Add to .gitignore
    .env.local
    
    ```
    

### **3. Perform Regular Backups**

- Backup the database and configuration files periodically to prevent data loss.
- Automate backups using Supabase's built-in tools or external solutions.

### **4. Document Changes**

- Maintain a changelog for configuration updates to simplify troubleshooting and audits.
